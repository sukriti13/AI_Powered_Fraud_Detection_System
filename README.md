# AI_Powered_Fraud_Detection_System
# 🔍 AI-Powered Fraud Detection & Explanation System

> An end-to-end ML + LLM pipeline that detects fraudulent credit card transactions and generates human-readable analyst notes — the way a real fraud analyst would reason.

---

## 📌 Overview

Financial fraud costs institutions billions annually. Traditional rule-based systems flag transactions but offer no reasoning — leaving analysts to manually investigate every alert.

This project solves that with a two-layer system:

- **ML Layer** — Random Forest classifier trained on real transaction data, handling severe class imbalance using SMOTE
- **LLM Layer** — LLaMA 3.1 (via Groq) generates a plain-English analyst note for every flagged transaction, explaining *why* it was flagged and recommending a next action
- **Dashboard** — Interactive Streamlit UI to analyze transactions in real time

The result mimics fraud operations tools used at companies like Stripe, Razorpay, and PayPal.

## 🏗️ Architecture

```
Raw Transaction Data (CSV)
         │
         ▼
┌──────────────────────┐
│  Preprocessing        │  ← StandardScaler, SMOTE for class imbalance
└──────────────────────┘
         │
         ▼
┌──────────────────────┐
│  Random Forest Model  │  ← Trained on 284K transactions
│  Fraud Score (0–1)    │  ← 94%+ recall on fraud class
└──────────────────────┘
         │
         ▼
┌──────────────────────┐
│  LLM Explanation      │  ← LLaMA 3.1 8B via Groq API
│  (Analyst Note)       │  ← Plain-English reasoning + recommendation
└──────────────────────┘
         │
         ▼
┌──────────────────────┐
│  Streamlit Dashboard  │  ← Live transaction analysis UI
└──────────────────────┘
```

## 📊 Dataset

**Credit Card Fraud Detection** — Kaggle
- 284,807 transactions | 492 fraudulent (0.17% — highly imbalanced)
- Features: 28 PCA-anonymized variables (V1–V28) + Amount + Time
- Link: https://www.kaggle.com/datasets/mlg-ulb/creditcardfraud

## ⚙️ Setup & Run

### 1. Clone the repo
```bash
git clone https://github.com/sukriti132/AI_Powered_Fraud-Detection-system.git
cd AI_Powered_Fraud-Detection-system
```

### 2. Create a virtual environment
```bash
python -m venv .venv
.venv\Scripts\activate        # Windows
source .venv/bin/activate     # Mac/Linux
```

### 3. Install dependencies
```bash
pip install -r requirements.txt
```

### 4. Get a free Groq API key
- Sign up at https://console.groq.com (no credit card needed)
- Create an API key and keep it handy

### 5. Download the dataset
- Download `creditcard.csv` from Kaggle
- Place it in the project root folder

### 6. Train the model (Phase 1)
```bash
python Phase1.py
```
This generates `fraud_model.pkl` in the project folder.

### 7. Test LLM explanation (Phase 2)
```bash
python Phase2.py
```

### 8. Run the dashboard
```bash
streamlit run dashboard.py
```
Open http://localhost:8501 — enter your Groq API key in the sidebar and click **Analyze Transaction**.
